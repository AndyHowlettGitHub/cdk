What is the CDK?

The CDK classes are Java utitility classes for ChemoInformatics and Computational chemistry, written in Java. They are developed constantly developed parallel to other projects that make use of them. They are a complete re-write of the CompChem classes that were the basis of JChemPaint, a Java Editor for 2D chemical structures, and of JMDraw, a Java package for the graphical layout of 2D chemical structures

A re-write was needed to in order to assure the widest possible usability. It is also intended to port JMol, a 3D molecular visualization tool, to the CDK.

The CDK classes support (or will support, as soon as ported from the old CompChem library):

  • Classes for Atoms, Molecules and other chemical concepts.

  • Utilities for importing and exporting chemical structures (Molecules or arrays of Atoms) using standard chemical files format.

  • Utility classes for handling chemical elements and isotopes.

  • A class that parses a molecular formula (a String of characters) and returns an array of Nodes.

  • An isomorphisms tester that compares to structures and returns if they are isomorph. I'm almost sure, that the implementation is not strictly valid, but it seems to work

  • A SMILES parser (hidden as a static method in compchem.ChemFileUtils), which so far only implements the SSMILES standard.

  • An implementation of John Figueras, "Ring Perception Using Breadth-First Search", J. Chem. Inf. Comput Sci. 1996, 36, 986-991.

Please check the existing API documentation for information on which classes are already implemented.

News

The CDK project is still in its initial stage. The people involved are those in the Chemoinformatics Group at the Max Planck Institute of Chemical Ecology in Jena, as well as Egon Willighagen from the University of Nijmegen, who is also involved with JChemPaint and many other Java Chemoinformatics programs, and Dan Gezelter, the initiator of JMol, a 3D viewer and analyzer for molecules .

Now, the rewrite of these CompChem/CDK library classes do somehow effect all of our projects. All of our code relies to some extend on the so-called CompChem classes for computational chemistry. I started this chemistry library when I started programming SENECA, my system for Computer Assisted Structure Elucidation. It turned out that some of the code had design flaws that prevented others, like the authors of (JMol) to use it. Egon, Dan and I now met last month at the University of Notre Dame, South Bend, Chicago, and redesigned the CompChem classes, calling the new project the "Chemistry Development Kit, CDK".

The development of all of our project will be organized via Sourceforge. Unfortunately, this system, as good as it is, turns out to be buggy and overloaded, but we'll stay with it, because this will not be so much of a problem as soon as the projects are established, we have the CVS access via SSH up'n running and can actually do some programming.

Our chemoinformatics and computational chemistry projects on Sourceforge are:

  1. The Chemistry Development Kit (CDK), which is our "new" library for Chemoinformatics and Computational Chemistry.

  2. http://www.sourceforge.net/projects/JChemPaint for JChemPaint development.

  3. http://www.sourceforge.net/projects/JMDraw for the development of JMDraw, which is the 2D rendering engine, also used in JChemPaint 1.0

  4. http://www.sourceforge.net/projects/compchem which contains the "old" compchem sources

Of course, at Notre Dame we were not able to do all the implementations. We decided to take some more care this time and start with UML. There are some UML diagrams on the CVS at http://www.sourceforge.net/projects/cdk which can be views with the program Linux program dia. Egon has moved them to ArgoUML, which is Java and can also be used under Windows.

The first step to do now is to create java classes for of all the existing UML diagrams. The second step is to fill them with life as it become clear what is needed while porting JChemPaint to them

Who develops the CDK?

The CDK is a joint venture of the JMol and JChemPaint developers who would like to see a general-purpose open-source library for computational chemistry and chemoinformatics. Check the developers list on the CDK pages.

How is the CDK developed

The CDK is mantained as a SourceForge project under http://www.sourceforge.net/projects/cdk. Please check the SourceForge site documentation for more. SourceForge gives us bugtracking, mailing lists, support manager and last not least CVS access.

How can I participate?

Check the SourceForge site documentation on how to access the CVS. If you actively want to contribute, please get a SourceForge accoutn, contact the project admin so that he can add you to the developers list
