What is the CDK?

The CDK classes are Java utitility classes for ChemoInformatics and Computational chemistry, written in Java. They are developed constantly developed parallel to other projects that make use of them. They are a complete re-write of the CompChem classes that were the basis of JChemPaint, a Java Editor for 2D chemical structures, and of JMDraw, a Java package for the graphical layout of 2D chemical structures

A re-write was needed to in order to assure the widest possible usability. It is also intended to port JMol, a 3D molecular visualization tool, to the CDK.

The CDK classes support (or will support, as soon as ported from the old CompChem library):

  • Classes for Atoms, Molecules and other chemical concepts.

  • Utilities for importing and exporting chemical structures (Molecules or arrays of Atoms) using standard chemical files format.

  • Utility classes for handling chemical elements and isotopes.

  • A class that parses a molecular formula (a String of characters) and returns an array of Nodes.

  • An isomorphisms tester that compares to structures and returns if they are isomorph. I'm almost sure, that the implementation is not strictly valid, but it seems to work

  • A SMILES parser (hidden as a static method in compchem.ChemFileUtils), which so far only implements the SSMILES standard.

  • An implementation of John Figueras, "Ring Perception Using Breadth-First Search", J. Chem. Inf. Comput Sci. 1996, 36, 986-991.

Please check the existing API documentation for information on which classes are already implemented.

Who develops the CDK?

The CDK is a joint venture of the JMol and JChemPaint developers who would like to see a general-purpose open-source library for computational chemistry and chemoinformatics. Check the developers list on the CDK pages.

How is the CDK developed

The CDK is mantained as a SourceForge project under http://www.sourceforge.net/projects/CompChem. Please check the SourceForge site documentation for more. SourceForge gives us bugtracking, mailing lists, support manager and last not least CVS access.

How can I participate?

Check the SourceForge site documentation on how to access the CVS. If you actively want to contribute, please get a SourceForge accoutn, contact the project admin so that he can add you to the developers list

What en Why?

This chapter explains what CDK is supposed to do, or, what it should be able to do. Since the CDK is a container for chemical data, the main requirements consists of the long list of chemical data it must be able to store. The long list is given below, in some generic hierarchy.

Since this requirement is not finished, do not expect it to be complete. Any additional requirements are welcomed. In fact, in important reason to write a requirement is to find out what the requirements really are.

Requirement 1: Storage of Chemical Data

Basic chemical 'things'

Basic chemical 'things' refer to chemical objects that every chemist uses, and which is not depending on the 'nature' of the chemist; wether he is a computational chemist, NMR specialist, an author, etc.

Required basic chemical objects are:

  • atoms (isotopes/elements)

  • bonds (aromatic/stereo)

  • orbitals (e.g. lone pairs)

More complex basic chemical 'things'

These objects are containers of other chemical objects:

  • molecules

  • salts

  • supramolecular structures

Other chemical 'things'

Other chemical 'things' refer to chemical objects that not every chemist uses. I.e., these objects are specialized.

Required basic chemical objects are:

  • magnetic shielding

  • molecular surfaces

Time dependant chemical 'things'

These objects are sequences of other chemical objects. A sequence might consist of a set of molecules, and depicting a vibration.

The time dependant chemical objects are:

  • animations

  • reactions
